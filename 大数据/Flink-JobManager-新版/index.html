# 概念

Flink 是一款开源的流处理框架，由 Apache Software Foundation 托管。📚它的核心设计思想是“批处理是流处理的一种特殊情况”，因此它提供了统一的流处理和批处理解决方案。

Flink 中的 JobManager 是一个非常重要的组件。💡你可以将其视为 Flink 作业的大脑。它负责接收提交的作业，调度和协调作业的执行，并在作业完成后收集结果。

这是一个 JobManager 在 Flink 作业中的角色的简单示例：

ExecutionEnvironment env = ExecutionEnvironment.getExecutionEnvironment();
DataSet<String> text = env.readTextFile("/path/to/file");
// Some transformations on the DataSet...
text.writeAsText("/path/to/output");
env.execute("My Flink Job");

在这段代码中，env.execute("My Flink Job") 将作业提交给了 JobManager，然后 JobManager 负责调度和执行这个作业。

  1. Flink 是一款开源的流处理框架，它的核心设计思想是“批处理是流处理的一种特殊情况”，因此它提供了统一的流处理和批处理解决方案。
  2. Flink 中的 JobManager 是一个非常重要的组件，可以将其视为 Flink 作业的大脑。它负责接收提交的作业，调度和协调作业的执行，并在作业完成后收集结果。

# 内部组件和交互

Flink JobManager 由几个主要组件组成：🔧

  1. Scheduler：负责在 TaskManager 上调度任务的执行。
  2. Checkpoint Coordinator：负责协调作业的检查点操作。
  3. Blob Store：用于存储作业的 JAR 文件和其他 BLOB 数据。

这些组件的交互可以使 JobManager 有效地协调 Flink 作业的执行。🔄 例如，当一个作业被提交到 JobManager 时，Scheduler 会分析作业的依赖关系，并决定任务的执行顺序。然后，它会将任务调度到可用的 TaskManager 上执行。在执行过程中，Checkpoint Coordinator 会定期触发检查点操作，以确保作业在发生故障时可以从最近的检查点恢复。

Scheduler 是 Flink JobManager 中的一个关键组件，负责管理作业的执行。它维护了作业的执行图，根据任务之间的依赖关系，决定任务的执行顺序。这意味着，如果你能有效地管理作业的依赖关系，可以优化任务的执行顺序，从而提高作业的执行效率。

Checkpoint Coordinator 负责协调作业的检查点操作。它定期触发检查点，将作业的状态保存到持久性存储中。这意味着，通过合理地设置检查点间隔，可以在保证作业可恢复性的同时，尽量减少检查点操作对作业性能的影响。

Blob Store 负责存储和分发作业的 JAR 文件和其他大对象（BLOB）数据。通过有效地使用 Blob Store，你可以确保所有的 TaskManager 都能获取到作业的代码和数据，从而提高作业的执行效率。 问题

# TaskManager

在 Flink 架构中，TaskManager 是执行具体任务的工作节点。每一个 TaskManager 都有一定数量的任务槽（Task Slot），这些任务槽用于并行执行任务。 TaskManager 的主要职责如下：

  1. 执行作业的具体任务
  2. 管理本地的资源
  3. 管理和维护状态和检查点
  4. 与 JobManager 和其他 TaskManager 通信

# 配置 JobManager 参数

Flink 提供了一个配置文件 flink-conf.yaml，可以在这个文件中设置各种参数，比如 JobManager 的内存大小，checkpoint 的间隔，等等。下面是一个示例：

3
4
5

8

# jobmanager 堆大小
jobmanager.heap.size: 1024m
# taskmanager 堆大小
taskmanager.heap.size: 4096m
# 每个 taskmanager 的任务槽数量
taskmanager.numberOfTaskSlots: 2
# 默认并行度
parallelism.default: 2

无需在主程序中手动读取 Flink 的配置。Flink 的配置由 Flink 运行时系统自动加载和处理。当提交作业时，Flink 的运行时系统会从默认的或者你指定的配置文件（例如：flink-conf.yaml）中读取配置信息。这些配置信息被读入到环境配置（Execution Environment）中，并被用来配置 JobManager，TaskManager，以及其他系统组件。 例如，你可以这样获取默认的并行度：

1
2

StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
int parallelism = env.getParallelism();

# JobManager 的故障恢复策略

Flink 的 JobManager 具有高可用性。这是通过一个称为 ZooKeeper 的分布式协调服务实现的。如果 JobManager 发生故障，Flink 可以自动选择一个新的 JobManager。这个过程是通过 ZooKeeper 中的领导者选举算法来完成的。 Flink 的作业在运行期间会周期性地在分布式存储系统中保存 checkpoint。当 JobManager 发生故障并被新的 JobManager 替换后，新的 JobManager 可以从最近的 checkpoint 中恢复作业的状态，并继续执行作业。

  2. 增加 JobManager JVM 堆大小
  3. 调整 Checkpoint 的频率
  4. 使用更好的硬件
